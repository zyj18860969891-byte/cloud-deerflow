## 🚀 部署步骤

### 步骤 1: 准备部署文件

确保项目根目录有以下文件:

```
deer-flow/
├── railway.json              ✅ 部署配置
├── Dockerfile                ✅ (或 docker/Dockerfile.backend)
├── docker-compose.yml        (可选)
├── backend/                  ✅ 后端代码
├── frontend/                 ✅ 前端代码
└── .env.example              ✅ 环境变量示例
```

### 步骤 2: 创建或更新 railway.json

在项目根目录创建 `railway.json`:

```json
{
  "$schema": "https://railway.app/railway.schema.json",
  "build": {
    "builder": "DOCKERFILE",
    "dockerfilePath": "docker/Dockerfile.backend"
  },
  "deploy": {
    "startCommand": "uvicorn app.main:app --host 0.0.0.0 --port $PORT",
    "healthcheckPath": "/health",

      "type": "ON_FAILURE"
    },
    "waitFor": ["postgresql", "redis"]

  "variables": {
    "ENVIRONMENT": "production",
    "LOG_LEVEL": "INFO"
  }
}
```

## 💾 完整部署脚本

创建文件 `scripts/deploy-railway.sh`:

```bash
#!/bin/bash

set -e

echo "🚀 开始 Railway 部署流程..."

# 1. 检查 Railway CLI
echo "📦 检查 Railway CLI..."
railway --version || {
    echo "❌ Railway CLI 未安装"
    echo "请运行: npm install -g @railway/cli"
    exit 1
}

# 2. 链接项目
echo "🔗 链接 Railway 项目..."
railway link --no-input

# 3. 配置环境变量
echo "⚙️  配置环境变量..."
railway variables set ALIPAY_APP_ID "2021006138604101"
railway variables set ALIPAY_PID "2088380691837603"
railway variables set ALIPAY_GATEWAY_URL "https://openapi.alipay.com/gateway.do"
railway variables set ENVIRONMENT "production"
railway variables set LOG_LEVEL "INFO"

# 如果有 ALIPAY_APP_PRIVATE_KEY 等敏感信息，通过 Dashboard 手动添加

# 4. 部署应用
echo "🚀 部署应用到 Railway..."
railway up

# 5. 等待部署完成
echo "⏳ 等待部署完成..."
sleep 30

# 6. 获取公域名
echo "📍 获取应用域名..."
DOMAIN=$(railway variables | grep RAILWAY_PUBLIC_DOMAIN | cut -d'=' -f2 | tr -d ' ')

# 7. 生成 NOTIFY_URL
NOTIFY_URL="https://${DOMAIN}/api/subscription/webhook-alipay"

echo ""
echo "✅ 部署完成！"
echo ""
echo "关键信息:"
echo "================================"
echo "公域名: https://${DOMAIN}"
echo "ALIPAY_NOTIFY_URL: ${NOTIFY_URL}"
echo "================================"
echo ""
echo "后续步骤:"
echo "1. 将以下 URL 配置到支付宝开放平台:"
echo "   ${NOTIFY_URL}"
echo ""
echo "2. 验证应用健康状态:"
echo "   curl https://${DOMAIN}/health"
echo ""
echo "3. 查看日志:"
echo "   railway logs --follow"
```

### 执行部署脚本 (Linux/Mac)

```bash
chmod +x scripts/deploy-railway.sh
./scripts/deploy-railway.sh
```

## 📍 获取 ALIPAY_NOTIFY_URL

### 自动获取方法

```bash
#!/bin/bash

# 1. 获取公域名
DOMAIN=$(railway variables | grep 'RAILWAY_PUBLIC_DOMAIN' | cut -d'=' -f2 | xargs)

# 2. 生成 NOTIFY_URL
NOTIFY_URL="https://${DOMAIN}/api/subscription/webhook-alipay"

# 3. 输出
echo "✅ ALIPAY_NOTIFY_URL=${NOTIFY_URL}"

# 4. 添加到 Railway 环境变量
railway variables set ALIPAY_NOTIFY_URL "${NOTIFY_URL}"

# 5. 验证
railway variables | grep ALIPAY_NOTIFY_URL
```

### 手动方法

1. **打开 Railway Dashboard**
   - 访问: https://railway.app
   - 进入您的 DeerFlow 项目
   - 选择 Backend Service

2. **查找 Public Domain**
   - 在 Settings 标签中找到 "Public Domain"
   - 复制域名 (例如: `deerflow-api.railway.app`)

3. **构造 NOTIFY_URL**
   ```
   https://[copied-domain]/api/subscription/webhook-alipay
   ```

4. **配置到支付宝平台**
   - 登录 https://open.alipay.com
   - 进入应用设置
   - 在 "异步通知地址" 配置此 URL

## 🚨 故障排除

### 问题 1: 部署失败 - Docker 镜像构建错误

**症状**:
```
❌ Build failed: Unable to build Dockerfile
```

**解决**:
```bash
# 1. 检查 Dockerfile 路径
ls docker/Dockerfile.backend

# 2. 本地测试构建
docker build -f docker/Dockerfile.backend -t deerflow:test .

# 3. 检查依赖
cd backend && pip install -r requirements.txt

# 4. 推送修复后重新部署
git push origin main
railway up
```

### 问题 2: 部署后应用崩溃 - /health 返回 500

**症状**:
```
curl: (22) HTTP error 400: Bad Request
```

**解决**:
```bash
# 1. 查看日志
railway logs --follow

# 2. 检查数据库连接
railway variables | grep DATABASE_URL

# 3. 检查数据库迁移是否执行
# Dockerfile 应包含: RUN alembic upgrade head

# 4. 检查环境变量是否正确加载
railway shell
echo $DATABASE_URL
echo $REDIS_URL
```

### 问题 3: 无法获取 Public Domain

**症状**:
```
railway domains (返回空)
```

**解决**:
```bash
# 1. 确认应用已成功部署
railway status

# 2. 检查服务是否在运行
railway service list

# 3. 重启服务
railway service restart

# 4. 通过 Dashboard 查看
# 项目 → Backend Service → Public Domain (通常在 Settings 中)
```
